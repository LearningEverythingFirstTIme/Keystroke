# 2026-03-27

## Phase 1 Complete: "Hello World" Keyboard Hook ✅

### What We Built
A console app that:
- Installs a system-wide low-level keyboard hook (`SetWindowsHookEx`)
- Listens for F12 key press
- Types "Hello World" into the focused app when triggered
- Uses a dedicated worker thread with queue to serialize typing operations

### Key Lessons Learned

**1. SendInput requires per-character delays**
- Initial approach sent all keystrokes at once → corruption ("Hello ddddd")
- Problem: Windows keystroke queue can't handle rapid programmatic input
- Solution: 15ms delay between each character
- 10ms was too fast (still corrupts), 20ms works but 15ms is the sweet spot

**2. Hook callback must return fast**
- Can't do heavy work (like SendInput) in the hook callback
- Solution: Queue requests, process in dedicated worker thread
- `BlockingCollection<string>` handles the queue safely

**3. P/Invoke struct layout is tricky**
- Initial INPUT struct was 32 bytes (wrong) → error 87 (invalid parameter)
- Fixed with explicit FieldOffset to get 40 bytes on 64-bit Windows
- Eventually switched to `InputSimulatorCore` library to avoid this complexity

**4. Task.Run causes SendInput interleaving**
- Each Task.Run creates a new thread
- SendInput calls from different threads can interleave at Windows level
- Solution: Single dedicated worker thread processes all typing

**5. _isTyping flag prevents queue pileup**
- Without flag, rapid F12 presses queued multiple requests
- Worker sets flag at start, clears at end
- Hook callback checks flag before queuing

### Architecture (Final)
```
Main Thread                    Worker Thread
    |                               |
    v                               |
SetWindowsHookEx                    |
    |                               |
HookCallback <---[keyboard]         |
    |                               |
Queue.TryAdd() -------------------->|
    |                          GetConsumingEnumerable()
    |                               |
    v                          Type each char
MessageLoop                    with 15ms delay
```
