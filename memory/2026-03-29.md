## Phase 3.1-3.6: Settings GUI + Panel Fix

### What We Built
1. **Settings Window** — dark themed WPF window with 3 card sections:
   - Prediction Engine: engine selector, API key, model dropdown
   - Completion Style: length preset (Brief/Standard/Extended/Unlimited), temperature slider, debounce slider
   - System Prompt: editable text box with Reset to Default button
   - All changes auto-save to config.json

2. **Completion Presets** — AppConfig now has CompletionPreset, Temperature, MaxOutputTokens
3. **System prompt editor** — user can fully customize the prompt sent to Gemini
4. **Panel shows only completion** — fixed panel to display Gemini's suggestion, not what user typed
5. **Fixed panel positioning** — anchored bottom-right instead of floating near cursor

### Bugs Fixed
- **Panel never visible**: OnBufferChanged was calling HideSuggestion() on every keystroke, hiding the panel before user could see it. Fix: removed HideSuggestion from OnBufferChanged.
- **Tab key inserting tab character**: Hook wasn't swallowing Tab. Fix: added SpecialKeyEventArgs.ShouldSwallow pattern, hook returns (IntPtr)1 when Tab is handled.
- **Gemini thinking tokens**: gemini-2.5-flash uses "thinking" tokens (90+ tokens) leaving only 1-4 for output. Fix: `thinkingConfig.thinkingBudget = 0` in API request.
- **Model deprecation**: gemini-2.0-flash deprecated. Use gemini-2.5-flash.
- **Dropdown text invisible**: ComboBoxItem style needed explicit Foreground color.
- **Panel positioning (3-in-1 fix)**: Replaced cursor-following position with fixed bottom-right anchor. Fixes: clipping, duplicate panels, intermittent invisibility.

### Panel Fix Details
- Removed `SetPosition(x, y)` and `CursorPositionHelper` dependency
- Panel uses `SystemParameters.WorkArea` to anchor bottom-right (respects taskbar)
- `Show()` called before `PositionBottomRight()` — WPF ignores Left/Top set before first Show()
- `OnRenderSizeChanged` re-anchors when content resizes
- EdgeMargin=20px, BottomMargin=16px

### Gotchas
- WPF ignores `Left`/`Top` set before the first `Show()` call — must Show first, then position
- The edit tool kept mangling files — when making complex changes, use `write` to rewrite entire files instead of `edit`
- WPF ComboBox dropdowns have white background by default — need explicit `ComboBoxItem` style with dark Foreground
- Gemini 2.5 Flash has thinking enabled by default — must explicitly disable with thinkingBudget=0
- `GetCaretPos` doesn't work cross-process — mouse position is the practical fallback for panel positioning
- DebounceTimer fires on a background thread — must use Dispatcher.Invoke for UI updates
